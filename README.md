# Workbench

Workbench is the internal codename for Ajinkya Gajbhiye's long-term personal digital ecosystem.

The first product inside Workbench is `ajinqya.com`: a personal website that should feel less like a portfolio and more like a living product. Over time, Workbench may also include software tools, experiments, plugins, AI projects, music, resources, and other products.

For now, this repository is intentionally documentation-first. The goal is to define the product before choosing implementation details.

## Current Status

Workbench is in the foundation phase.

- No application framework has been added yet.
- No Next.js code exists yet.
- Product, content, architecture, and design decisions are being collected in `docs/`.

## Working Principles

- Start with intent before implementation.
- Build systems instead of one-off pages.
- Treat everything as structured, indexable content.
- Prefer durable structure over short-lived cleverness.
- Keep the site personal, useful, enjoyable, and maintainable.
- Document decisions as they become real.
- Discuss meaningful architecture tradeoffs before committing to them.
