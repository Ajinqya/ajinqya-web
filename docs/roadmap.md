# Roadmap

This roadmap is intentionally provisional. It captures the likely phases for Workbench before detailed implementation begins.

## Phase 0: Foundation

Status: In progress

- Establish repository documentation.
- Define vision, content model, design principles, and architecture direction.
- Record open decisions and placeholders.
- Avoid committing to a frontend framework prematurely.

## Phase 1: Product Definition

Status: TODO

- Define version-one scope.
- Identify core content types.
- Draft information architecture.
- Choose tone, visual direction, and editorial principles.
- Define how exploration, discovery, and "something new every visit" should work.
- Decide what must ship first versus what can wait.

## Phase 2: Technical Foundation

Status: TODO

- Choose application framework and hosting model.
- Define content storage strategy.
- Set up formatting, linting, testing, and CI.
- Establish environment and deployment workflow.
- Create first implementation skeleton.

## Phase 3: First Public Version

Status: TODO

- Build homepage.
- Build core content routes.
- Publish initial content.
- Build an initial index or discovery surface.
- Add metadata, SEO, accessibility, analytics, and feed support as appropriate.
- Deploy a stable first version.

## Phase 4: Ecosystem Expansion

Status: TODO

- Add richer project pages.
- Add experiments or tools.
- Add search, tags, collections, or graph-like navigation.
- Improve authoring workflow.
- Explore integrations with other personal systems.
- Expand beyond `ajinqya.com` into other Workbench products when ready.
